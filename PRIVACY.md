# Privacy Notice

**RTN Student Self-Report Intake App**
Last updated: March 2026

## Summary

This application **does not collect, transmit, or store any personal data**. All information entered into the form remains entirely within the user's web browser and is never sent to any external server, database, or third party.

## Data Handling

| Action | What happens |
|--------|-------------|
| Filling out the form | Data is held in browser memory (React state) only |
| "Save as File" | Exports a `.json` file to the user's local device |
| "Print / Save PDF" | Uses the browser's native print dialog |
| Closing the browser tab | All entered data is permanently lost |
| Refreshing the page | All entered data is permanently lost |

## What This App Does NOT Do

- ❌ Send data to any server or API
- ❌ Use cookies, localStorage, or sessionStorage
- ❌ Include analytics, tracking pixels, or telemetry
- ❌ Use third-party scripts that collect data (no Google Analytics, no Meta Pixel, etc.)
- ❌ Store any data persistently in the browser
- ❌ Require user accounts, logins, or authentication

## External Resources

The only external resources loaded are **Google Fonts** (Baloo 2 and Nunito), which are subject to [Google's Privacy Policy](https://policies.google.com/privacy). No user data is sent to Google as part of font loading. If this is a concern, the fonts can be self-hosted.

## Use With Minors

This tool is designed for **clinician-administered use**. A licensed professional (SLP, reading specialist, educator, or psychologist) should be present when a minor is providing responses. The tool should not be distributed directly to minors for unsupervised completion.

## FERPA and HIPAA Considerations

Because this application does not transmit or store data on any server, it does not independently create FERPA or HIPAA compliance obligations. However:

- **Clinicians are responsible** for handling the exported JSON files and printed PDFs in accordance with applicable privacy regulations (FERPA, HIPAA, state laws)
- Exported files should be stored in secure, compliant systems (e.g., encrypted EHR, secure file storage)
- Printed copies should be handled per your organization's records management policies
